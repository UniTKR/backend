# ADR-0001: Spring Modulith 기반 모듈러 모놀리스로 시작한다

## 상태

Accepted

## 날짜

2026-05-08

## 맥락

UniT는 대학생 전용 중고거래 플랫폼이다. 초기 단계에서는 빠른 개발 속도, 낮은 운영 복잡도, 단순한 배포 구조가 중요하다.

다만 기능 영역은 사용자, 학교 인증, 판매글, 거래, 채팅, 후기, 신고, 관리자 기능처럼 명확히 나뉜다. 향후 트래픽이나 운영 요구가 커질 경우 일부 영역을 별도 서비스로 분리할 가능성도 있다.

## 결정

초기 아키텍처는 MSA가 아니라 Spring Modulith 기반 모듈러 모놀리스로 구성한다.

## 이유

- 하나의 Spring Boot 애플리케이션으로 실행, 배포, 운영을 단순하게 유지할 수 있다.
- 모듈 경계를 코드 수준에서 검증할 수 있다.
- 초기 개발 속도를 유지하면서 향후 서비스 분리 가능성을 남길 수 있다.
- 트랜잭션과 데이터 일관성 관리가 MSA보다 단순하다.
- 현재 단계에서는 서비스별 독립 배포보다 명확한 도메인 분리가 더 중요하다.

## 결과

- UniT 백엔드는 하나의 애플리케이션과 하나의 포트로 실행된다.
- 모듈은 패키지 경계와 Spring Modulith 규칙으로 관리한다.
- 모듈 간 직접 의존은 제한하고 공개된 서비스 계약을 통해 협력한다.
- 향후 특정 모듈의 독립 배포 필요성이 명확해지면 서비스 분리를 재검토한다.

## 대안

- 전통적인 단일 모놀리스: 초기 개발은 빠르지만 모듈 경계가 쉽게 흐려질 수 있다.
- MSA: 서비스 독립성과 확장성은 좋지만 초기 운영 복잡도, 배포 복잡도, 분산 트랜잭션 부담이 크다.
